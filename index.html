Identity Anchor Protocol (IAP)

IAP provides cryptographic certificate issuance services for persistent autonomous agents, enabling verifiable identity and continuity over time.

What it is

Identity & continuity primitives

Minimal primitives: Identity Anchor, Continuity Certificates, and Lineage declarations. Certificates are digitally signed and verifiable offline using published registry keys.

How it works

Pay-per-issuance, verifiable offline

Customers pay per certificate issuance. Once issued, a certificate becomes a verifiable cryptographic artifact. See the quickstart below for a typical developer flow. 

pip install iap-sdk
iap-agent init
iap-agent anchor issue
iap-agent flow run
iap-agent verify certificate.json
